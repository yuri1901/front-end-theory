# Детальний конспект: Огляд ARIA (Accessible Rich Internet Applications)

## Загальна інформація

- **Опис**: ARIA (Accessible Rich Internet Applications) — це набір атрибутів, які додаються до HTML-елементів для покращення доступності веб-додатків, особливо для користувачів зі скрінрідерами.
- **Контекст**: Використовується для опису семантики та поведінки кастомних елементів, динамічного контенту та складних інтерфейсів, які не підтримуються стандартним HTML.
- **Основні компоненти**:
  - **Атрибути**:
    - `role`: Визначає тип елемента (наприклад, `button`, `dialog`).
    - `aria-*`: Описує стан або властивості (наприклад, `aria-label`, `aria-hidden`, `aria-disabled`).
  - **Категорії ролей**: Структурні, віджети, орієнтири, живі регіони.
  - **Сумісність**: Інтегрується з HTML5 семантикою.
- **Особливості**:
  - ARIA доповнює, але не замінює семантичний HTML.
  - Використовується, коли стандартні HTML-елементи не передають достатньої інформації.
  - Потребує тестування зі скрінрідерами (NVDA, VoiceOver).

## Приклади коду

### 1. Використання `role` і `aria-label`

Позначення кастомної кнопки.

```html
<div role="button" aria-label="Закрити" onclick="closeModal()">X</div>
```

#### Результат:

Скрінрідер оголошує елемент як кнопку з описом "Закрити".

### 2. Позначення стану з `aria-disabled`

Кнопка в неактивному стані.

```html
<button role="button" aria-disabled="true">Відправити</button>
```

#### Результат:

Скрінрідер повідомляє, що кнопка неактивна.

### 3. Живий регіон із `aria-live`

Оновлення повідомлення.

```html
<div role="alert" aria-live="assertive">Помилка: невірний пароль!</div>
```

#### Результат:

Скрінрідер миттєво оголошує повідомлення.

### 4. Прихований елемент із `aria-hidden`

Приховування декоративного контенту.

```html
<img src="decor.png" alt="" aria-hidden="true" />
```

#### Результат:

Скрінрідер ігнорує елемент.

## Особливості для співбесід — Питання та відповіді

### Що таке ARIA?

- **Відповідь**: Набір атрибутів для покращення доступності веб-додатків, що додають семантику та стани до HTML.
  ```html
  <div role="button">Клік</div>
  ```

### Коли використовувати ARIA?

- **Відповідь**: Для кастомних елементів, динамічного контенту або коли семантичний HTML недостатній.
  ```html
  <div role="dialog" aria-label="Вікно">...</div>
  ```

### Яка різниця між `aria-label` і `aria-labelledby`?

- **Відповідь**: `aria-label` задає текстовий опис, `aria-labelledby` посилається на ID іншого елемента.
  ```html
  <button aria-label="Закрити">X</button>
  <button aria-labelledby="label-id">X</button>
  <span id="label-id">Закрити</span>
  ```

### Чи може ARIA замінити семантичний HTML?

- **Відповідь**: Ні, ARIA доповнює HTML; краще використовувати семантичні елементи, коли можливо.
  ```html
  <button>Кнопка</button>
  <!-- Краще, ніж div з role="button" -->
  ```

## Поради

- **Семантика перша**: Використовуйте ARIA лише тоді, коли семантичний HTML недостатній.
- **Тестування**: Перевіряйте з NVDA, VoiceOver або JAWS для коректної роботи.
- **Мінімізуйте ARIA**: Надмірне використання може заплутати скрінрідери.
- **Документація**: Звертайтеся до WAI-ARIA специфікації на MDN або W3C.
- **Фокус**: Забезпечте клавіатурну навігацію для елементів із ARIA-ролями (наприклад, `tabindex`).

## Додаткові деталі

- **Ролі**:
  - Орієнтири: `banner`, `navigation`, `main`.
  - Віджети: `button`, `checkbox`, `dialog`.
  - Живі регіони: `alert`, `log`, `status`.
- **Атрибути**:
  - `aria-label`: Текстовий опис.
  - `aria-hidden`: Приховує елемент від скрінрідерів.
  - `aria-live`: Позначення динамічного контенту.
  - `aria-controls`: Посилається на керований елемент.
- **Сумісність**: Підтримується у всіх сучасних браузерах і скрінрідерах.
- **Обмеження**: Неправильне використання ARIA може погіршити доступність.
