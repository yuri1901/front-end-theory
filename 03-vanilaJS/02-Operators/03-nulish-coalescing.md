# Оператор нульового злиття в JavaScript (`??`)

## Загальна інформація

- **Опис**: Оператор нульового злиття (`??`) повертає правий операнд, якщо лівий є `null` або `undefined`. Використовується для встановлення значень за замовчуванням, коли потрібно ігнорувати лише "нульові" значення.
- **Синтаксис**:
  ```js
  leftOperand ?? rightOperand;
  ```
- **Особливості**:
  - Додано в ECMAScript 2020.
  - На відміну від `||`, який повертає правий операнд для всіх falsy значень (`false`, `0`, `""`, `null`, `undefined`, `NaN`), `??` перевіряє лише `null` і `undefined`.
  - Повертає значення лівого операнда, якщо він не є `null` або `undefined`.
  - Часто використовується для безпечного доступу до даних і значень за замовчуванням.
- **Контекст**: Застосовується в ситуаціях, де `0`, `""` або `false` є валідними значеннями, і потрібно уникнути їх заміни.

## Приклади коду

### Базове використання

```js
let value = null;
console.log(value ?? "default"); // "default"

let validValue = 0;
console.log(validValue ?? "default"); // 0
```

### Порівняння з `||`

```js
let emptyString = "";
console.log(emptyString || "default"); // "default" (|| вважає "" falsy)
console.log(emptyString ?? "default"); // "" (?? зберігає "")

let zero = 0;
console.log(zero || 42); // 42
console.log(zero ?? 42); // 0
```

### Використання в об’єктах

```js
let user = { name: null };
console.log(user.name ?? "Guest"); // "Guest"
console.log(user.age ?? 18); // 18 (age is undefined)
```

### Комбінація з `&&`

```js
let data = { config: null };
console.log(data.config && data.config.setting); // null
console.log(data.config ?? data.defaultConfig); // data.defaultConfig
```

## Особливості для співбесід — Питання та відповіді

### Чим відрізняється `??` від `||`?

- `||` повертає правий операнд, якщо лівий є falsy (`false`, `0`, `""`, `null`, `undefined`, `NaN`).
- `??` повертає правий операнд, лише якщо лівий — `null` або `undefined`.
  ```js
  console.log(false || "default"); // "default"
  console.log(false ?? "default"); // false
  ```

### Чи можна комбінувати `??` з `&&` або `||`?

- Так, але потрібно використовувати дужки через різний пріоритет операторів (`&&` і `||` мають вищий пріоритет, ніж `??`).
  ```js
  let x = null;
  console.log((x ?? "default") && true); // "default"
  console.log(x ?? ("default" && true)); // true
  ```

### Коли використовувати `??`?

- Для значень за замовчуванням, коли `0`, `""` або `false` є валідними значеннями.
  ```js
  let count = 0;
  console.log(count ?? 10); // 0 (count не є null/undefined)
  ```

### Чи є `??` частиною логічних операторів?

- Технічно це окремий оператор, але він часто використовується в подібних контекстах для умовної логіки.

## Поради

- **Використовуй `??` замість `||` для значень за замовчуванням**: Це запобігає помилковій заміні `0`, `""` чи `false`.
- **Додавай дужки для ясності**: При комбінації з `&&` чи `||` використовуй дужки, щоб уникнути помилок із пріоритетом.
- **Перевіряй `null`/`undefined` явно**: `??` ідеально підходить для безпечного доступу до даних.
- **Уникай надмірного використання**: Для складних умов використовуй `if...else` для кращої читабельності.
